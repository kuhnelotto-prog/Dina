# Аудит indicators_calc.py

## Общая информация
- Файл: `indicators_calc.py`
- Назначение: вычисление технических индикаторов, свечных паттернов, FVG (Fair Value Gap) и Liquidity Sweep.
- Класс: `IndicatorsCalculator`
- Зависимости: pandas, numpy, ta (Technical Analysis Library)

## Критические проблемы

### 1. Потенциальная ошибка индексации в `_check_patterns`
- В строках 83-85 вычисляются `last`, `prev`, `pprev` как `len(df) - 1`, `-2`, `-3`.
- Если `len(df) < 3`, то `pprev` становится отрицательным (например, -1 при len=2).
- В строках 109-110 используется условие `if pprev >= 0`, но `pprev` может быть -1, что делает условие ложным, и `fvg_bull`/`fvg_bear` становятся `False`. Это корректно, но может ввести в заблуждение.
- **Рекомендация**: Добавить явную проверку `if len(df) >= 3` в начале метода.

### 2. Неэффективное вычисление индикаторов
- Метод `compute` вычисляет индикаторы для всего DataFrame с помощью библиотеки `ta`, но использует только последние значения.
- Для больших исторических данных это приводит к избыточным вычислениям.
- **Рекомендация**: Рассчитать индикаторы только для необходимого окна (например, последние 30 свечей) или использовать кэширование.

### 3. Обработка недостаточных данных
- При `len(df) < 30` возвращается словарь с `"error": "insufficient_data"`, но остальные поля заполняются значениями по умолчанию (RSI=50, ATR=0 и т.д.).
- Вызывающий код может проигнорировать ошибку и использовать некорректные значения.
- **Рекомендация**: Возвращать `None` или вызывать исключение, либо явно документировать поведение.

### 4. Деление на ноль в `volume_ratio`
- Если `vol_sma.iloc[last] == 0`, возникает деление на ноль. Сейчас есть проверка `if vol_sma.iloc[last] > 0 else 1.0`.
- Однако нулевой SMA объема маловероятен, но возможен на малых таймфреймах с нулевым объемом.
- **Рекомендация**: Оставить как есть, но добавить комментарий.

### 5. Упрощенная логика FVG
- FVG определяется как `low.iloc[last] > high.iloc[pprev]` (бычий) и `high.iloc[last] < low.iloc[pprev]` (медвежий).
- Это упрощение, пропускающее среднюю свечу. Классический FVG требует трех свечей: свеча 1, гэп, свеча 2.
- **Рекомендация**: Пересмотреть логику в соответствии с общепринятым определением FVG.

### 6. Сортировка данных
- Метод `_ensure_ascending` сортирует только по столбцу `timestamp`, если он существует.
- Если timestamp отсутствует, данные могут быть в неправильном порядке, что повлияет на индикаторы.
- **Рекомендация**: Добавить сортировку по индексу, если timestamp отсутствует, или требовать наличие timestamp.

## Второстепенные проблемы

### 1. Дублирование кода
- `last`, `prev`, `pprev` вычисляются в `compute` (строки 53-55) и затем снова в `_check_patterns` и `_check_sweep`.
- **Рекомендация**: Передавать эти индексы как параметры или вычислять один раз в `compute` и передавать в методы.

### 2. Отсутствие аннотаций типов для возвращаемых значений
- Методы `_check_patterns` и `_check_sweep` возвращают `dict`, но типы значений не указаны.
- **Рекомендация**: Добавить `-> Dict[str, bool]` и т.п.

### 3. Магические числа
- В коде используются "магические" числа: 30, 14, 20, 2, 9, 21.
- **Рекомендация**: Вынести в константы класса или конфигурацию.

### 4. Возможность улучшения производительности
- Индикаторы MACD и Bollinger Bands вычисляются, но не все результаты используются (например, только последние значения полос).
- **Рекомендация**: Если нужны только последние значения, использовать `ta` функции, которые возвращают скаляр для последнего элемента.

### 5. Обработка NaN в индикаторах
- Проверки `if not pd.isna(...)` избыточны, если данные гарантированно не содержат NaN.
- **Рекомендация**: Убедиться, что входные данные очищены от NaN, либо оставить проверки.

## Заключение
Код в целом работоспособен, но содержит несколько потенциальных проблем, которые могут привести к некорректным результатам в edge-случаях. Рекомендуется устранить критические проблемы, особенно связанные с логикой FVG и обработкой недостаточных данных.

**Приоритеты исправлений**:
1. Исправить логику FVG.
2. Добавить проверку длины df в `_check_patterns`.
3. Улучшить обработку недостаточных данных.
4. Вынести магические числа в константы.
