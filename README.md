# Дина — торговый бот для Bitget Futures (USDT-M)

**Дина** — это автономный торговый бот для фьючерсов Bitget (USDT-M), написанный на Python.  
Проект находится в активной разработке. Бот использует мульти‑таймфреймовый анализ, адаптивные веса сигналов (LearningEngine), жёсткий риск‑менеджмент и может работать в полностью автономном режиме.

## Архитектура

- **Два независимых модуля**: `Дина-Лонг` и `Дина-Шорт` (шорт имеет более строгие фильтры).  
- **Оркестратор** управляет обоими модулями, распределяя капитал и согласуя сигналы.  
- **Мульти‑таймфреймовость**: анализ на 15m, 1h, 4h.  
- **Набор монет**: 10 наиболее ликвидных фьючерсных пар (BTC, ETH, SOL и др.).  
- **Плечо**: по умолчанию 3x (в перспективе адаптивное до 10x).

## Реализованные компоненты

- **main.py** — точка входа, инициализация всех модулей.  
- **bitget_executor.py** — исполнение ордеров, сухой запуск (dry‑run), управление позициями.  
- **risk_manager.py** — дневные лимиты, лимиты позиций, общая экспозиция, проверка корреляции (базовая).  
- **learning_engine.py** — адаптивные веса сигналов; холодный старт с `DEFAULT_WEIGHTS` и минимальным числом сделок (`MIN_TRADES_TOTAL=50`).  
- **telegram_bot.py** — уведомления с приоритетами (critical/high/normal/info), ночной режим (в разработке).  
- **orchestrator.py** — управление ботами (лонг/шорт).  
- **signal_builder.py** — мульти‑таймфреймовая агрегация, FVG, Liquidity Sweep.  
- **event_bus.py** — шина событий для асинхронной коммуникации.  
- **trade_log.py** — журнал сделок с комиссиями и метаинформацией.  
- **position_sizer.py** — динамический расчёт размера позиции (Kelly, ATR, просадка).  
- **performance_attribution.py** — учёт P&L по источникам сигналов.  
- **validate_dina.py** — интеграционный тест для проверки работоспособности.  

**Ключевые механики** уже работают:
- **Trailing stop / profit** (4 этапа на ATR).  
- **Reconciliation** при рестарте: восстановление позиций с проверкой SL; аварийный SL = 1.5 × ATR при отсутствии.  
- **Retry‑логика** с экспоненциальной задержкой для надёжности API.  
- **ExecutionGuard** (allowlist символов, rate limits, максимальный размер позиции).  
- **Dry‑run режим** — торговля без отправки реальных ордеров.  
- **EventBus**, **DataFeed** (заглушка), **SignalBuilder** — основа для потоковой обработки данных.

## Что в планах (приоритеты)

### 1. SafetyGuard (критично)
Независимый watchdog‑процесс, который мониторит позиции напрямую через API биржи. При аномалиях (ATR‑шок, быстрая просадка, расширение спреда) принудительно закрывает позиции. Включает circuit breaker (остановка торговли при просадке >15%) и операторский keepalive (если 7 дней нет команды — плавное закрытие).

### 2. Завершение Дина‑Шорт
Конфигурация для отдельного запуска (параметризация или второй экземпляр). Шорт должен использовать более жёсткие фильтры сигналов.

### 3. Точный учёт комиссий
Добавить в `TradeRecord` поле `commission_filled` — реальная комиссия из ответа биржи, а не расчётная. Это важно для налоговой отчётности (особенно для немецкой юрисдикции).

### 4. Расширенная корреляционная проверка
В `risk_manager._check_correlated_exposure` добавить группы: L1, L2, DeFi, AI, Gaming, Infra, Meme. Лимиты на одновременные позиции в секторе. Для BTC — отдельная поправка (коэффициент 0.5).

### 5. Ночные уведомления
Доработать `telegram_bot.py` с поддержкой `SILENT_HOURS` и чёткой градацией приоритетов, чтобы критичные сообщения доходили даже ночью, а остальные накапливались для утренней сводки.

### 6. Автономность и резервные каналы
- `TELEGRAM_OFFLINE_TOLERANCE` — продолжение работы при недоступности Telegram.  
- Email fallback для критических алертов.  
- Жёсткие лимиты в коде: `MAX_DAILY_LOSS_PCT`, `MAX_TOTAL_DRAWDOWN`, `MAX_POSITION_AGE_HOURS`.

### 7. Режимная классификация (MarketRegimeDetector)
Простой детектор на основе ATR, объёма, волатильности BTC (CALM / VOLATILE / CRISIS) с автоматической подстройкой параметров риска.

### 8. Оптимизация `bitget_executor.py`
Разбить большой файл (~38KB) на отдельные модули: `api_client`, `order_manager`, `trailing_manager`, `state_recovery` — для удобства поддержки.

### 9. Упрощённый портфельный VaR
Расчёт Value at Risk по открытым позициям раз в 5–10 минут; при превышении лимита уменьшение размера новых сделок.

### 10. Долгосрочные улучшения
- GMM / продвинутая классификация режимов.  
- LightGBM для макро‑факторов.  
- Оркестратор + легковесные CoinAgent (масштабирование на 30–75 монет).  
- Веб‑дашборд (FastAPI).  
- Полноценный data_feed с вебсокетами.

## Важные архитектурные решения

- **Плечо**: по умолчанию 3x, в будущем адаптивное до 10x.  
- **VPS**: Contabo SG1 (Сингапур) — после первой стабильной версии.  
- **GitHub** — основное хранилище кода.  
- **Оператор keepalive**: если 7 дней нет команды от оператора, бот плавно закрывает позиции и переходит в режим ожидания.  
- **Автономность**: Telegram может быть недоступен, бот продолжает работать.  
- **Налоговая отчётность**: логируется фактическая комиссия из ответа биржи, а не расчётная.

## Быстрый старт

1. Скопировать `.env.example` в `.env` и заполнить ключи Bitget и Telegram.  
2. Установить зависимости: `pip install -r requirements.txt`.  
3. Запустить валидацию: `python validate_dina.py`.  
4. Запустить в dry‑run: `python main.py`.  
5. Для перехода на реальные деньги установить `DRY_RUN=false` и убедиться, что все защитные механизмы (SafetyGuard и др.) готовы.
